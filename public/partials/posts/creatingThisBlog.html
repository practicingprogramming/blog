The first post is, naturally, about creating this website.

There are a lot of blogging platforms out there, but writing things by myself is the very purpose of this blog. So, I decided to make my own blogging platform from scratch. It doesn't have to be a real platform though: it should just work for this particular website.

These are the minimal requirements:

  • Must be able to host static articles.
  • Must be able to show those static articles in a feed.
  • Must have "About" page.

Desired but not necessary for launch:

  • Ability to leave comments.
  • Convenient publishing of new material.
  • Analytics.

To satisfy the minimal requirements itâ€™s enough for it to be completely static (HTML + JS + CSS, no backend). All I need to do is to write a bunch of HTML and host it somewhere. I chose to host in on AWS S3 as it looks like a good option, plus I'm quite familiar with it.

Technologies I chose to use:

  • AWS S3 + Route53 for hosting.
  • node + expressjs for running it locally.
  • Grunt for building.
  • Bootstrap for styling.
  • Angular.

Overall, it was pretty straightforward. Hosting a website on S3 is described in much details here. The only thing that required some coding effort was displaying all posts at once on the front page. But that wasn't difficult either.

Things I've learned:

  • How to host a website on S3.
  • How to register a domain and point it at the website with Route53.
  • How to setup Angular and Grunt. I've participated in projects using them, but had never done the initial setup myself before.

What I should do in the future:

  • Add unit tests. Usually I'm all for test-driven development. But in this case I have probably 10 lines of code, so I decided it was not even worth bothering.
  • Add some sort of analytics. Perhaps I should use one of the services analyzing S3 access logs. It's a little strange that S3 itself does not provide this functionality.
  • Enable SSL. Not a big deal at the moment because it's static, so there are no interactions to secure.
  • Use CloudFront for distribution.

Conclusions:

  • AWS is a great option for hosting static websites.
